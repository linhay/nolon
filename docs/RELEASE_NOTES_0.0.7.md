# Release Notes 0.0.7

## [0.0.7] - 2026-01-26

本次更新带来了全新的新手引导流程、深度重构的设置页面、以及更强大的 Git 仓库集成能力。

### 新特性

- **全新新手引导 (Onboarding)**：
  - 引导式初次配置体验。
  - 自动检测并预设已安装的 AI Coding Agents。
  - 支持 Liquid 风格的视觉动态效果。
- **深度重构的设置中心**：
  - 全新的双栏式外观设计。
  - **外观设置**：支持系统/浅色/深色模式切换。
  - **多语言支持**：新增中（简/繁）、日、英多语言支持（需重启生效）。
  - **取消链接策略**：新增解除仓库链接时的文件处理策略（询问/保留/删除）。
  - **工作区管理**：可视化展示当前工作区根路径。
  - **开机启动**：一键管理应用自启动。
- **增强的 Git 技能仓库集成**：
  - 自动识别仓库名称与托管服务商（GitHub/GitLab/Bitbucket）。
  - **多路径支持**：允许在同一个仓库中添加多个技能存放路径。
  - **自动同步**：显著优化了 Git 仓库的拉取与内容解析效率。
- **全新的技能浏览器 (Remote Skills Browser)**：
  - 支持远程技能的按列表与网格视图切换展示。
  - 深度集成仓库目录树浏览。

### 改进与优化

- **核心架构升级**：
  - 重构了 `SkillParser` 和 `SkillInstaller`，显著提升解析稳定性。
  - 引入了 `AppCommandState` 统一管理应用全局状态（如设置面板显示）。
- **孤立技能 (Orphaned Skills) 处理**：
  - 自动识别并标记未正确安装的孤立技能。
  - 提供一键迁移与修复功能。
- **UI/UX 细节优化**：
  - 改进后的侧边栏支持直接在分类下编辑或移除仓库。
  - 优化了弹窗与表单的交互反馈。
- **国际化翻译**：
  - 依托最新脚本自动化补全了多语言翻译缺失。

### 修复

- 修复了 URL Scheme `nolon://` 触发不稳定导致无法快速添加仓库的问题。
- 修复了某些情况下 `SKILL.md` 的 YAML 元数据无法正确读取的问题。
- 修复了 Sparkle 自动更新配置中的遗留安全警告。
